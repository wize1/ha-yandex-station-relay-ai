# Yandex Station Relay AI (Кузя)

Кастомная интеграция Home Assistant, которая передаёт речь между **умными
колонками с Яндекс Алисой (Яндекс Станция)** и **ИИ-агентом по вашему выбору** —
через настраиваемое триггерное слово.

> «Алиса, попроси **Кузю** рассказать, что нового в мире» → отвечает ваш ИИ →
> Алиса произносит ответ.

Триггерное слово («Кузя») — это имя, которое вы дадите навыку. Спрашивайте что
угодно после него; фраза передаётся либо **агенту беседы Home Assistant** (OpenAI,
Anthropic, Google, Ollama, …), либо напрямую на **OpenAI-совместимый / свой
endpoint**, и ответ возвращается через Алису.

---

## Как это работает

```
«Алиса, попроси Кузю <фраза>»
        │
        ▼  Яндекс Диалоги  ──HTTPS POST──►  вебхук HA (эта интеграция)
                                                  │  удаление триггерных слов
                                                  ▼
                                          ИИ-бэкенд (на ваш выбор)
                                                  │
        Алиса произносит ответ  ◄──JSON-ответ─────┘
```

Два транспорта, работающих вместе:

1. **Навык Яндекс Диалогов (основной вход).** Ваш экземпляр HA размещает вебхук
   навыка. Яндекс изначально понимает `«Алиса, попроси <навык> <фраза>»`,
   передаёт фразу в вебхук и произносит то, что вебхук вернёт.
2. **TTS Яндекс Станции (вывод / запасной вариант при медленном ответе).** Яндекс
   Диалоги ждут ответ ~3 секунды, но LLM часто медленнее. Когда ответ медленный,
   навык отвечает короткой фразой *«Секунду, думаю…»*, а затем **упреждающе
   произносит полный ответ через настроенную колонку** (используя TTS
   `media_player` из [AlexxIT/YandexStation](https://github.com/AlexxIT/YandexStation)).
   Колонка не настроена? Тогда навык предложит сказать *«дальше»*, чтобы услышать
   ответ из очереди на следующем шаге.

Длинные ответы (> 1024 символов — лимит Яндекса) автоматически разбиваются на
части и зачитываются по частям.

---

## Требования

- Home Assistant **2024.12+**.
- Доступ к вашему HA по **публичному HTTPS с валидным сертификатом** (Nabu Casa
  Cloud, Cloudflare Tunnel или обратный прокси). Яндекс не обращается к endpoint
  без доверенного сертификата.
- Аккаунт Яндекса для создания навыка Диалогов (бесплатно).
- *(Необязательно, рекомендуется)* установленная интеграция **YandexStation** и
  ваша колонка, добавленная как `media_player` — включает упреждающие голосовые
  ответы и сервис `speak`.

## Установка

### HACS (рекомендуется)
1. HACS → ⋮ → **Пользовательские репозитории** → добавьте этот репозиторий как
   **Интеграцию**.
2. Установите **Yandex Station Relay AI**, затем перезапустите Home Assistant.

### Вручную
Скопируйте `custom_components/yandex_station_relay_ai/` в папку
`config/custom_components/` вашего HA и перезапустите.

## Настройка

### 1. Добавьте интеграцию в HA
**Настройки → Устройства и службы → Добавить интеграцию → Yandex Station Relay AI.**

- Задайте имя (например, `Кузя`).
- *(Необязательно)* вставьте **Skill ID** вашего навыка, чтобы отклонять запросы
  от любых других навыков.
- На последнем экране показывается ваш **URL вебхука** — скопируйте его. Он
  выглядит так: `https://<ваш-домен>/api/webhook/<random-id>`

### 2. Создайте навык Яндекс Диалогов
На [dialogs.yandex.ru/developer](https://dialogs.yandex.ru/developer):

1. **Создать диалог → Навык в Алисе.**
2. **Имя / имя активации:** ваше триггерное слово, например `Кузя`. Именно это
   заставляет работать `«Алиса, попроси Кузю …»`.
3. **Backend:** выберите **Webhook** и вставьте URL из шага 1.
4. Установите **язык навыка — русский**.
5. Сохраните. Навык можно использовать на ваших привязанных устройствах в режиме
   **черновика/тестирования** — для личного использования модерация не нужна.
   (Публикация делает `попроси Кузю` доступным везде; тестовый режим работает на
   устройствах вашего аккаунта.)

### 3. Выберите ИИ-бэкенд
На карточке интеграции → **Настроить**:

- **Агент беседы Home Assistant** — направляйте запрос любому настроенному агенту
  Assist (OpenAI, Anthropic, Google Generative AI, Ollama, свой). Оставьте *ID
  агента беседы* пустым для агента по умолчанию. Это «агент по общему выбору».
- **OpenAI-совместимый / свой endpoint** — вызывайте любой API `/chat/completions`
  напрямую: OpenAI, OpenRouter, LM Studio, vLLM, OpenAI-endpoint Ollama или свой
  сервер. Укажите базовый URL, ключ API (если есть), модель и системный промпт.

Здесь же настраивается: **Яндекс Станция** для упреждающих ответов, триггерные
слова для удаления, фразы приветствия / ожидания / ошибки и таймауты.

### 4. Попробуйте
> «Алиса, попроси Кузю придумать рифму к слову дом»

## Сервисы

### `yandex_station_relay_ai.ask`
Передаёт текст настроенному ИИ и возвращает его ответ (с возможностью произнести
вслух). Полезно в автоматизациях или как «путь через YandexStation».

```yaml
action: yandex_station_relay_ai.ask
data:
  text: "Какая погода обычно в июне в Лондоне?"
  speak: true                                   # произнести на колонке
  target: media_player.yandex_station_kitchen   # необязательно; по умолчанию — настроенная колонка
response_variable: result
# result.text -> ответ ИИ
```

### `yandex_station_relay_ai.speak`
Произносит произвольный текст на колонке — без вызова ИИ, только TTS.

```yaml
action: yandex_station_relay_ai.speak
data:
  text: "Ужин готов"
  target: media_player.yandex_station_kitchen
```

## Поведение и тонкая настройка

- **Таймаут первого ответа** (по умолчанию 2.5 с): сколько ждать ИИ перед
  запасной фразой *«думаю…»*. Держите меньше ~3 с — Яндекс отбрасывает более
  медленные ответы.
- **Держите ответы короткими и пригодными для произнесения.** Системный промпт по
  умолчанию просит одно-два предложения без markdown. Настройте под свой бэкенд.
- **Триггерные слова** вроде `кузя`/`кузю` удаляются из начала фразы; связки вроде
  `попроси`, `спроси`, `алиса` удаляются всегда.
- **Контекст по сессиям:** каждый диалог с Алисой получает свою ветку беседы,
  поэтому уточняющие вопросы сохраняют контекст.

## Устранение неполадок

| Симптом | Вероятная причина |
|---|---|
| Алиса: «навык не отвечает» | ИИ медленнее таймаута и колонка не настроена — настройте колонку для упреждающих ответов или уменьшите `max_tokens`. |
| Яндекс не может сохранить вебхук | HA недоступен по публичному HTTPS с валидным сертификатом. |
| Ничего не происходит | Несовпадение URL вебхука или задан фильтр `skill_id`, который не совпадает. Проверьте логи HA (`yandex_station_relay_ai`). |
| Упреждающий ответ не произносится | `tts_entity` не задан или это не рабочий `media_player` YandexStation. |

---

## Ограничения

- Захват произвольной фразы **требует навыка Диалогов** (публичный HTTPS-вебхук).
  Сам по себе YandexStation не может передать HA произвольный распознанный текст —
  здесь он используется для **вывода** (TTS) и сервисов `ask`/`speak`.
- Упреждающие ответы произносятся на единственной настроенной колонке, а не
  автоматически на том устройстве, к которому обратился пользователь.
